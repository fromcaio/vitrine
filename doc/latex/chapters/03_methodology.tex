\chapter{Metodologia}\label{cap:Metodologia}

Este capítulo detalha a metodologia de pesquisa e os processos de engenharia de software aplicados no desenvolvimento do sistema Vitrine durante o período de iniciação científica. A abordagem adotada busca unir o rigor acadêmico à aplicação prática de tecnologias modernas para solucionar o problema da publicação acadêmica independente.

\section{Classificação da Pesquisa}

Quanto à sua \textbf{natureza}, esta pesquisa é classificada como \textbf{aplicada}, visto que objetiva a criação de um artefato tecnológico — um sistema de gerenciamento de conteúdo (CMS) — destinado a resolver o problema prático da presença digital docente. Quanto aos \textbf{objetivos}, o trabalho possui caráter \textbf{descritivo} e \textbf{exploratório}. É exploratório na fase de investigação de tecnologias como o framework Next.js e o paradigma Jamstack, e descritivo ao detalhar as etapas de modelagem, arquitetura e requisitos da solução.

\section{Procedimentos Técnicos}

Para a condução do estudo, foram adotados os seguintes procedimentos:

\begin{itemize}
    \item \textbf{Revisão Bibliográfica}: O foco recaiu sobre arquiteturas web, bancos de dados portáteis (SQLite) e padrões de design para interfaces de usuário, fornecendo o embasamento necessário para as decisões arquiteturais do projeto \cite{ref7}.
    \item \textbf{Desenvolvimento Experimental e Incremental}: O método baseia-se no desenvolvimento de um protótipo funcional através de um processo \textbf{iterativo e incremental} \cite{Sommerville2011}. Esta abordagem permitiu que funcionalidades complexas fossem validadas de ponta a ponta a cada incremento realizado.
\end{itemize}

\section{Etapas do Desenvolvimento}

O processo de construção do Vitrine foi estruturado em quatro macroetapas, fundamentadas nos processos clássicos de engenharia de software \cite{Sommerville2011}:

\begin{enumerate}
    \item \textbf{Levantamento de Requisitos}: Identificação das necessidades dos usuários (professores e desenvolvedores), resultando na definição de Requisitos Funcionais (RF) e Não-Funcionais (RNF) \cite{Sommerville2011}.
    \item \textbf{Modelagem de Sistemas}: Utilização da linguagem UML e DBML para visualização da lógica do sistema, incluindo Diagramas de Casos de Uso e Modelagem de Dados (ER) \cite{Sommerville2011}.
    \item \textbf{Prototipagem}: Criação de protótipos de \textbf{baixa fidelidade} para validar a usabilidade do editor de Markdown e do painel administrativo antes da implementação final \cite{Sommerville2011}.
    \item \textbf{Desenvolvimento Incremental (Sprints)}: Implementação do código-fonte utilizando Next.js e Prisma ORM, abrangendo desde a persistência no SQLite até a renderização no front-end e configuração de cache (ISR) \cite{nextjs_docs, prisma_docs}.
\end{enumerate}

\section{Requisitos do Sistema}

A especificação dos requisitos foi a base para o desenvolvimento incremental da plataforma \cite{Sommerville2011}:

\begin{itemize}
    \item \textbf{RF01 - Autenticação}: Acesso restrito via Auth.js para segurança no gerenciamento.
    \item \textbf{RF02 - Estrutura Hierárquica}: Suporte a documentos e coleções aninhadas.
    \item \textbf{RF03 - Ordenação}: Atributo de ordem relativa entre elementos irmãos.
    \item \textbf{RF04 - Personalização}: Interface para alteração de branding e cores via variáveis CSS.
\end{itemize}
