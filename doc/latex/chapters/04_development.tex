\chapter{Desenvolvimento do Sistema}\label{cap:devSistema}

Neste capítulo, detalham-se as etapas de construção do sistema Vitrine, abrangendo desde a concepção visual por meio de protótipos até a implementação técnica da arquitetura baseada em Next.js e SQLite.

\section{Arquitetura Técnica}

A arquitetura do Vitrine foi planejada sob o paradigma Jamstack, visando desacoplar a gestão de dados da camada de apresentação \cite{nextjs_docs}. 
\begin{itemize}
    \item \textbf{Front-end}: Desenvolvido com o framework \textbf{Next.js}, utilizando renderização híbrida para otimizar o SEO e a experiência do usuário \cite{nextjs_docs}.
    \item \textbf{Persistência}: Utilização do \textbf{Prisma ORM} em conjunto com o \textbf{SQLite}. A escolha do SQLite como banco de dados baseado em arquivo é estratégica para garantir a portabilidade e reduzir a complexidade de manutenção para o usuário final \cite{prisma_docs}.
    \item \textbf{Autenticação}: Implementada via \textbf{Auth.js}, garantindo um sistema de login simplificado e seguro para o gerenciamento de conteúdos.
\end{itemize}

\section{Implementação das Funcionalidades Centrais}

A construção seguiu um modelo incremental, onde cada funcionalidade foi desenvolvida de forma completa, integrando banco de dados e interface \cite{Sommerville2011}.

\subsection{Editor de Escrita Científica}
O editor foi implementado para suportar as necessidades específicas da academia, incluindo o realce de sintaxe e a renderização de fórmulas matemáticas complexas. O sistema utiliza um motor de Markdown que converte o conteúdo em HTML de alto desempenho no momento da build ou via \textit{Incremental Static Regeneration} (ISR) \cite{nextjs_docs}.

\subsection{Sistema de Cores e ISR}
Uma das inovações técnicas do projeto é a manipulação de variáveis CSS em tempo de execução \cite{nextjs_docs}. As preferências de cores são persistidas no banco de dados SQLite e aplicadas globalmente. Para evitar a perda de performance, utilizou-se a ISR, que reflete as alterações estéticas instantaneamente sem a necessidade de uma nova compilação total do projeto (\textit{build}) \cite{nextjs_docs}.

\subsection{Hierarquia e Portabilidade de Dados}
A estrutura de dados foi modelada para suportar documentos e coleções aninhadas através de uma auto-relação na tabela de conteúdos do Prisma \cite{prisma_docs}. Isso garante que o usuário tenha total controle sobre a organização do seu conhecimento, mantendo a posse integral do arquivo de banco de dados para futuras migrações \cite{prisma_docs}.
