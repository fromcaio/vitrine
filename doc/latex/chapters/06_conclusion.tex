\chapter{Conclusão}\label{cap:conclusao}

Este relatório detalhou o ciclo de desenvolvimento do sistema \textbf{Vitrine}, uma plataforma CMS \textit{open source} voltada para a democratização da publicação acadêmica independente. Ao longo do período de iniciação científica, entre setembro de 2025 e fevereiro de 2026, foi possível converter o desafio da dependência de plataformas proprietárias em uma solução funcional que prioriza a soberania do autor sobre sua produção intelectual.

\section{Resultados Alcançados}

Os objetivos propostos foram integralmente atingidos através de uma abordagem de desenvolvimento incremental \cite{Sommerville2011}. O sistema consolidou o equilíbrio entre modernidade tecnológica e portabilidade ao utilizar o framework \textbf{Next.js} e o banco de dados \textbf{SQLite} \cite{nextjs_docs, prisma_docs}. Entre as principais conquistas técnicas, destacam-se:

\begin{itemize}
    \item A implementação de um editor \textit{markdown} capaz de renderizar com precisão elementos essenciais da escrita científica, como fórmulas em \textit{LaTeX} e blocos de código com realce de sintaxe;
    \item O desenvolvimento de uma interface de personalização visual dinâmica, permitindo que o usuário gerencie a estética do site via variáveis CSS persistidas em banco de dados, com reflexão instantânea através de \textit{Incremental Static Regeneration} (ISR) \cite{nextjs_docs};
    \item A garantia da perenidade do conteúdo digital, uma vez que a arquitetura baseada em arquivos portáteis assegura que o pesquisador detenha a posse integral de seus dados \cite{prisma_docs}.
\end{itemize}

\section{Limitações e Desafios}

Apesar dos avanços, o projeto encontrou fronteiras delimitadas pelo prazo de execução. A modelagem da interface restringiu-se a protótipos de baixa fidelidade, focando na funcionalidade e usabilidade em detrimento de um refinamento visual mais exaustivo \cite{Sommerville2011}. Além disso, o sistema de autenticação via \textit{Auth.js}, embora seguro e funcional para o gerenciamento básico, ainda pode ser expandido para suportar múltiplos níveis de permissão em cenários colaborativos.

\section{Considerações Finais e Trabalhos Futuros}

A experiência de desenvolver o Vitrine reafirmou a premissa de que sistemas de informação devem atuar como facilitadores da disseminação de conhecimento, e não como obstáculos técnicos \cite{KeRainerJr2019}. O projeto entrega uma base sólida e moderna que elimina o trabalho repetitivo de configuração estrutural, permitindo que o foco permaneça na produção de conteúdo qualificado.

Como propostas para a continuidade e expansão do projeto, pretende-se:
\begin{itemize}
    \item Produzir documentação técnica detalhada e tutoriais em vídeo demonstrando o processo de clonagem do repositório e \textit{deployment} na plataforma Vercel;
    \item Lançar o site oficial do projeto, denominado \textbf{Vitrine}, que servirá como repositório central para os tutoriais, documentação e instâncias de demonstração;
    \item Explorar a integração com repositórios de dados abertos para a citação automática de publicações;
    \item Expandir a biblioteca de componentes visuais, permitindo maior customização da interface;
    \item Implementar a renderização nativa de gráficos para funções de uma variável (gráficos 2D).
\end{itemize}

O Vitrine encerra este ciclo de iniciação científica não apenas como uma ferramenta técnica, mas como um manifesto em prol da autonomia digital e da democratização do saber científico na rede.
